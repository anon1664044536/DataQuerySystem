## 智能问数系统开发日志

### 0.0.1

**系统初始化与可行性预研**

*   **技术栈构建**：搭建基于 Java + Spring Boot + DashScope (通义千问) 的语义解析基础框架。
*   **MQL 定义**：制定了基于 JSON 的中间表示协议 (MQL)，用于解耦自然语言意图与物理 SQL 语法。
*   **对比实验**：实现了与 NatSQL 的对照性研究脚本，验证了在复杂嵌套查询下使用中间表示层的架构优势。

### 0.0.2

**电力行业领域知识库与基准建设**

*   **数据集构建**：针对 PMS (电力管理系统) 业务场景，手动构造了覆盖资产、备件、项目、组织架构四大领域的 100 道高质量题目（Questions + Golden SQL）。
*   **评测基座**：开发了 `DatasetEvaluator` 评测引擎及 `ReferenceBuilder` 基准答案生成工具。
*   **原型验证**：完成了第一版基于双阶段 (NL2MQL -> MQL2SQL) 的原型机开发，建立了 Execution Accuracy 的自动化度量体系。


### 0.1.0

**第一版生产级语义解析流水线 (PMS 专用版)**

本次更新完成了从“原型机”向“工业级”管线的深度演进，重点解决了微服务异构数据库环境下的逻辑幻觉与库表寻址问题。

#### 1. 架构升级：引入三阶段解析管线
*   **NL2MQL (IR 提取)**：由直接生成 SQL 转为生成中间表示 (JSON)，锁定业务意图。
*   **Schema Linker (子结构裁剪)**：[新增] 根据 MQL 实时提取核心 DDL 结构。强制注入 `USE <database>;` 指令，彻底解决了多库环境下“表不存在”的报错。
*   **MQL2SQL (精准生成)**：[优化] 将原始问题、MQL 以及 Schema Linker 裁剪出的 DDL 三方加权，生成最终 SQL。

#### 2. “零幻觉”安全加固
*   **精准前缀法则**：强制要求模型根据注入的 `USE` 指令补全表前缀（如 `middleground_ast.xxx`），禁止随意猜测 `pms.` 前缀。
*   **被动关联防御**：严禁模型在 DDL 缺失外键的情况下盲目 JOIN 机构表，杜绝了无中生有的虚假关联。
*   **自动过滤丢弃**：模型现在能识别 DDL 缺失列，并自动丢弃 MQL 中无效的 `filters`（如时间列缺失），不再生成 `1=0` 这种逻辑死锁 SQL。

#### 3. 运维与工程化优化
*   **交互式控制台**：新增 `run.bat` 和 `run.ps1`脚本，支持一键切换 Web 运行、PMS 评测、Spider 评测及 Python 统计报告。
*   **可视化 Web UI**：更新了聊天界面，支持实时查看 **Sub-Schema (裁剪后的 DDL)** 展示和管线各阶段耗时打点。
*   **安全屏障**：建立 `application.properties.example` 规范，实现敏感 API Key 的物理隔离与 Git 保护。
